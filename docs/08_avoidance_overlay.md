# 08 段B 回避オーバーレイ(idea3移植)— 正しいが Prime ではほぼ不発

> 統合設計 docs/06 段B。finance で唯一頑健だった idea3(毒性ファイナンス=行使価額修正条項付
> 新株予約権/MSCB を出した銘柄を回避)を本リポに移植し、Prime ユニバースでの実効果を測った。
> 実装: `src/ingest/financing_events.py`(分類+yanoshin前進取込+finance DB seed)/ `src/overlay/avoidance.py`
> (除外マスク+赤旗)/ `scripts/run_avoidance_overlay.py`。実施日: 2026-06-24。

## 方法

- **イベント取込**: finance の `financing_events`(2023-09〜2026-02、**1,243件**=Tier A 567 / Tier B 676)を
  seed 移植(yanoshin 再取得45分を回避=無料APIに優しい。前進取込モジュールも用意)。
- **分類**: タイトルのみで Tier A(行使価額修正/MSCB)/ Tier B(普通の第三者割当)を判定(finance `classify_financing` 移植)。
- **Prime 交差**: yanoshin 5桁コード→ティッカー(例 76030→7603.T)で、本リポの Prime 価格データ(1,565銘柄)と突合。
- **イベントスタディ**: 発表翌営業日始値入り・自ユニバース中央値で市場中立・分割除外(`compute_event_returns`)、+5/20/60日。
- **除外効果**: 段Aの正直シグナル群(7,253件)を任意ロング母集団とみなし、Tier A 発表後60日以内の銘柄を除外して
  ドローダウン/最悪値が改善するか。

## 結果①(本命の発見): **毒性ファイナンスの88%は Prime の外にある**

| | 全市場(dedup後) | うち Prime |
|---|---|---|
| Tier A(行使価額修正/MSCB) | 286件 / 158銘柄 | **33件 / 24銘柄(12%)** |

- **Tier A の88%は Standard/Growth など Prime 外の小型・不振銘柄**。行使価額修正型ファイナンスは
  「資金繰りに窮した小型株」の現象で、Prime 大型はほとんど発行しない。

## 結果②: Prime 内 Tier A は劣後を示さない(N過少)

Prime の33件・市場中立後中央値:

| ホライズン | 中央値 | 平均 | 深下落(<-10%) |
|---|---|---|---|
| +5日 | +0.21% | -1.01% | 9% |
| +20日 | +0.64% | -1.29% | 21% |
| +60日 | **+1.75%** | +0.08% | **27%** |

- finance(全市場・中立後 +60日 ~-3〜-4%)と違い、**Prime 部分集合では中央値はむしろプラス**。
  左裾(深下落27%)は残るが **N=33 では確証不能**。Prime 内では回避シグナルが立たない。

## 結果③: 除外オーバーレイは Prime シグナルにほぼ当たらない

- 段Aの正直シグナル7,253件のうち、Tier A 除外で外れたのは **わずか4件(0.06%)**。
- 除外してもDD・最悪値は不変(母集団の最悪 -61.81% は赤旗銘柄ですらない)。
- → **Prime 限定の本システムでは、回避オーバーレイは事実上の不発**。

## 判定: 層Bは「正しく動くが、ユニバースが間違っている」

- idea3 の移植・分類・イベントスタディ・除外ロジックは**正しく動作**(finance を独立再現する基盤が完成)。
- しかし**Prime には毒性ファイナンスがほぼ無い**(88%が外)ため、Prime 限定では保険として機能しない。
- 赤旗表示は**残す価値あり**(無料・コスト0・稀に当たれば左裾を1件削れる)が、損益インパクトは Prime では限定的。

## 段A+段B を貫く構造的結論: **Prime は土俵が違う**

2フェーズが独立に同じ場所を指した:
- **段A**: Prime 大型にテクニカル超過リターンは無い(裁定済み。+0.35%はルックアヘッド)。
- **段B**: 唯一の検証済み保険(毒性ファイナンス回避)も Prime にはほぼ存在しない(88%が外)。

→ **エッジも地雷も、機関が来られない Standard/Growth の小型にある**(finance の中核仮説=「個人の優位は
機関が行かない極小型にしかない」と完全に一致)。**本システムが Prime に固執する限り、攻め(段A)も守り(段B)も
空振りする。**

## 次アクション(エビデンス駆動の方向転換)

1. **ユニバースを Standard/Growth 小型へ拡張**(最重要・新方向)。そこでは ①回避オーバーレイ(段B)が
   本当に効く(Tier A の大半が居る・finance で中立後-3〜-4%実証)②小型の非効率/流動性プレミアムの検証余地がある。
   - データ: JPX マスタの Standard/Growth を `ticker_master` に追加(無料)。価格は yfinance(廃止銘柄の生存バイアスは要注記)。
   - まず「小型ユニバースで段Aの正直BTを再実行」+「Tier A 除外オーバーレイのDD改善」を測る。
2. 赤旗オーバーレイ(段B)は**全ユニバースで常時表示**として残す(無料の保険)。
3. 段C(レジーム・リスク管理)は小型ユニバース確定後に重ねる(小型ほどDD管理の価値が高い)。

```
uv run python scripts/run_avoidance_overlay.py   # seed→Prime交差→イベントスタディ→除外効果
```
`--finance-db` で finance の screener.db パスを指定(既定 D:\work\finance\data\db\screener.db)。
